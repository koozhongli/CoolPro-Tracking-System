Functional Requirements (FR):

1.	Registration: System shall allow coaches to register new players with personal details.
   
2.	Profile Management: System shall allow players to update their contact information.
   
3.	Performance Entry: System shall allow coaches to input match statistics (e.g., goals, assists, speed) for specific players.
   
4.	Tournament Creation: System shall allow admins to create new tournament events with dates and venues.
   
5.	Roster Management: System shall allow coaches to assign players to specific tournament squads.
    
6.	Attendance Tracking: System shall record player attendance for training sessions.
    
7.	Performance Analysis: System shall generate a graphical report of a player's progress over time.
    
8.	Injury Logging: System shall allow medical staff to log player injuries and expected recovery dates.
    
9.	Fee Management: System shall track monthly training fee payments.
    
10.	Search: System shall allow users to search for players by name or category (e.g., U-12, U-18).
    
11.	Notifications: System shall send email alerts for upcoming tournaments.
    
12.	Dashboard: System shall display a summary of upcoming events on the admin dashboard.

Non-Functional Requirements (NFR):

1.	Scalability: The database must support up to 10,000 player records without performance degradation.
   
2.	Security: All user passwords must be hashed using bcrypt before storage.
   
3.	Availability: The system must be available 99.9% of the time during business hours (8 AM - 8 PM).
   
4.	Usability: The UI must be responsive and accessible on both tablets (for coaches on the field) and desktops.
   
5.	Performance: The system must load the player dashboard in under 2 seconds.
    
6.	Data Integrity: The system must perform daily automated backups of the player database.
